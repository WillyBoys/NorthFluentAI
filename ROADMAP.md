# ROADMAP.md

## Purpose

This roadmap defines the phased growth plan for NorthFluent AI so the project stays realistic in the short term and scalable in the long term.

## Guiding Principle

Build a strong, polished first version before expanding scope.

## Phase 1: Class Project MVP

### Goal
Create a clean educational website that introduces NorthFluent AI and delivers useful beginner content plus one interactive feature.

### Required Deliverables
- Brand identity direction
- Core website structure
- 4 to 5 educational pages plus homepage
- Prompt Improver Tool
- Consistent design system
- Responsive layout

### Recommended Deliverables
- Reusable components
- Content driven page data where helpful
- Strong navigation and page to page flow
- Basic polish for demo readiness

### Success Criteria
- The platform is easy to understand
- The design feels intentional and polished
- The content is clear for beginners
- The Prompt Improver Tool works
- The site can be demoed confidently

## Phase 2: Expanded Learning Platform

### Goal
Turn the MVP into a more complete beginner AI learning hub.

### Possible Additions
- More lessons
- More examples
- More use case pages
- Downloadable resources
- Beginner quiz
- AI tools page
- Better interactive learning elements

### Focus
Expand depth without losing beginner friendliness.

## Phase 3: Niche Learning Paths

### Goal
Create targeted paths for specific audiences.

### Possible Tracks
- AI for students
- AI for job seekers
- AI for creators
- AI for small business

### Focus
Make content more tailored while preserving the core NorthFluent AI voice.

## Phase 4: Monetization

### Goal
Introduce sustainable growth options once the platform has clear value and audience fit.

### Potential Monetization Paths
- Premium guides
- Prompt packs
- Courses
- Newsletter
- Workshops
- Consulting or speaking

### Rule
Do not design the MVP around monetization. Earn trust first.

## Suggested Build Sequence for Now

1. Set up base project structure
2. Build layout shell with navigation and footer
3. Build homepage
4. Build AI Basics page
5. Build Prompt Better page
6. Build Strengths and Limits page
7. Build About page
8. Build Prompt Improver Tool page
9. Polish styling and responsiveness
10. Review for clarity and scope discipline

## Scope Control Checklist

Before adding anything new, ask:
- Is this necessary for the MVP?
- Does this make the site more useful for beginners?
- Will this improve clarity?
- Can this be added later instead?

If it does not clearly improve the first version, postpone it.

## Risks to Avoid

- Trying to build too many features at once
- Writing content that is too advanced for beginners
- Making the interface feel too technical
- Spending too much time on backend work not needed for the MVP
- Adding pages without a clear role in the learning journey

## Near Term Recommendation

Focus on building a polished educational front end first. A strong static or mostly static MVP is the best path for speed, clarity, and demo quality.
